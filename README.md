# 🎭 ИГРА В АРТЁМА

Многопользовательская игра на дедукцию. Один из игроков — тайный **Артём**: ему не показывают вопрос, и он должен блефовать. Остальные по ответам пытаются вычислить, кто же притворяется.

## Как запустить

Нужен установленный **Node.js** (версия 16+). Скачать: https://nodejs.org

1. Распакуй папку с игрой.
2. Открой терминал в этой папке и выполни:

```bash
npm install
npm start
```

3. Сервер напишет адрес, например `http://localhost:3000` — открой его в браузере.

## Как играть с друзьями (в одной Wi-Fi сети)

- Узнай локальный IP компьютера-сервера:
  - **Windows:** `ipconfig` → строка «IPv4-адрес» (например `192.168.1.50`)
  - **macOS/Linux:** `ifconfig` или `ip addr` (ищи `192.168.x.x`)
- Друзья на телефонах/ноутбуках в той же сети заходят на `http://192.168.1.50:3000`
- Каждый вводит ник → один создаёт лобби → остальные входят по 4-значному коду.

> Чтобы играть через интернет (не одна сеть), сервер нужно развернуть на хостинге
> (Render, Railway, Glitch, VPS и т.п.) — код полностью совместим.

## Правила

1. Все вводят ники в лобби. Нужно **минимум 3 игрока** (лучше 5+).
2. Один игрок жмёт **«Я АРТЁМ»** — забирает тайную роль. После этого кнопка у всех меняется на **«ГОТОВ»**.
3. Когда все нажали «Готов» — матч начинается. Кто Артём, знает только он сам.
4. На каждый вопрос отвечает **случайный** игрок. Остальные смотрят, как он печатает ответ вживую.
5. Когда отвечает **Артём** — ему вопрос **не показывается**, он должен выкручиваться. Остальные видят вопрос и его ответ.
6. Ники скрыты под **мерцающими символами** — игроки различаются только по номеру (#1, #2…).
7. Любой может нажать **«Голосование · кикнуть игрока»**. Все голосуют за номер (или «пропустить»).
   - Выгнали Артёма → **победа команды**.
   - Выгнали невиновного → он выбывает, игра продолжается.
8. Игра идёт, **пока не кончатся вопросы**. Если Артём дожил — **победа Артёма**.

## Файлы

- `server.js` — сервер (Express + Socket.io): лобби, ходы, лайв-печать, голосование.
- `public/index.html` — клиент: весь интерфейс, дизайн и логика экрана.
- `package.json` — зависимости.

## Настройки (в `server.js`)

- `MIN_PLAYERS` — минимум игроков (по умолчанию 3).
- `MAX_PLAYERS` — максимум (по умолчанию 10).
- `MATCH_QUESTIONS` — сколько вопросов длится матч (по умолчанию 15).
- `QUESTIONS` — банк вопросов (про Dota/CS2, компьютеры и про мир), добавляй свои.
- `PORT` — порт (по умолчанию 3000).
