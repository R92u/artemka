# 🚀 ГАЙД: запуск «Игры в Артёма» на Render

Здесь два способа: **через `render.yaml`** (проще, всё настроится само) и **вручную**.
Плюс — что делать с ошибками. Выбирай способ А.

---

## 0. Что нужно

- Аккаунт на **GitHub** (репозиторий у тебя уже есть — `artemka`).
- Аккаунт на **Render** → https://render.com (войти можно через GitHub).

---

## 1. Залей свежие файлы в GitHub

В архиве лежат файлы (в т.ч. новый `render.yaml`). Их надо положить в репозиторий **с сохранением структуры папок** — то есть `index.html` должен остаться внутри папки `public/`.

1. Распакуй архив `igra-v-artema.zip` у себя на компьютере.
2. Зайди в свой репозиторий на GitHub → кнопка **Add file → Upload files**.
3. **Перетащи мышкой** в окно загрузки СОДЕРЖИМОЕ распакованной папки:
   - `server.js`
   - `package.json`
   - `render.yaml`
   - `.gitignore`
   - `README.md`
   - `DEPLOY.md`
   - папку `public` целиком (внутри неё `index.html`)
   > GitHub при перетаскивании папки сохраняет структуру — это важно.
4. Внизу нажми **Commit changes**.
5. Проверь в репозитории: файл `public/index.html` должен лежать ИМЕННО внутри `public`, а не в корне.

> ⚠️ Не загружай папку `node_modules` — она не нужна, Render сам поставит зависимости.

---

## 2А. Деплой через Blueprint (рекомендую)

Так Render возьмёт настройки из `render.yaml`, и ничего не надо вбивать руками.

1. На Render: **New → Blueprint**.
2. Подключи/выбери репозиторий `artemka`.
3. Render покажет, что нашёл сервис `igra-v-artema`. Нажми **Apply / Create**.
4. Жди сборку. В логах должно появиться:
   ```
   🎭  ИГРА В АРТЁМА запущена
   ```
5. Готово — открывай выданную ссылку вида `https://igra-v-artema.onrender.com`.

---

## 2Б. Деплой вручную (если Blueprint не используешь)

1. На Render: **New → Web Service**.
2. Выбери репозиторий `artemka`.
3. Заполни поля:
   | Поле | Значение |
   |------|----------|
   | **Language / Runtime** | `Node` |
   | **Branch** | `main` |
   | **Root Directory** | *(пусто)* |
   | **Build Command** | `npm install` |
   | **Start Command** | `npm start` |
   | **Instance Type** | `Free` |
4. Нажми **Create Web Service** и дождись строки `ИГРА В АРТЁМА запущена` в логах.

> ❗️ Важно: это должен быть именно **Web Service**, а НЕ **Static Site**.
> Static Site не запускает Node-сервер → именно поэтому раньше была ошибка «Not Found».

---

## 3. Как играть

1. Открой ссылку сервиса — это и есть адрес игры.
2. Скинь её друзьям (работает с любого устройства, интернет есть — и норм).
3. Каждый вводит ник → один жмёт **«Создать лобби»** → остальные **«Подключиться»** по 4-значному коду.
4. Все жмут **«Я АРТЁМ»** (готовность). Когда готовы все (минимум 3) — старт.

---

## 4. Если что-то не так

**Видишь «Not Found»**
→ Сервис создан как *Static Site*. Удали его и пересоздай как **Web Service** (см. 2Б) или через Blueprint (2А).

**Видишь «Bad Gateway» / 502 / пустую страницу**
→ Сервер не стартовал. Открой вкладку **Logs** в Render и посмотри ошибку. Чаще всего:
- забыли `Build Command: npm install`;
- структура папок сломана (`index.html` оказался не в `public/`).

**Страница долго грузится (30–60 сек), крутится**
→ Это норма для бесплатного тарифа: после ~15 минут без активности сервис «засыпает» и при первом заходе просыпается. Дальше работает быстро.

**Игроки не видят друг друга / лобби не создаётся**
→ Значит соединение Socket.io не идёт. На Render Web Service WebSocket работает из коробки — проверь, что открыт именно `https://...onrender.com` (а не локальный адрес), и что сервис в статусе **Live**.

**Обновил код — как передеплоить?**
→ Просто залей изменённые файлы в GitHub (как в шаге 1). При `autoDeploy: true` Render пересоберёт сам. Или нажми **Manual Deploy → Deploy latest commit**.

---

## 5. Полезное

- Поменять вопросы / минимум игроков — в `server.js` (массив `QUESTIONS`, константы `MIN_PLAYERS`, `MAX_PLAYERS`).
- Своё доменное имя можно подключить в настройках сервиса Render (Settings → Custom Domains).
